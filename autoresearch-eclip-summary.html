Autoresearch on an Old Research Idea — Summary

Originally published by ykumar.me on March 22, 2026. Summarized here for reference.

Overview

The author revisited their old research code for eCLIP — a CLIP-style vision-language model augmented with expert spatial attention via Gaussian heatmaps — and handed it to Claude Code to run unsupervised ML experimentation using Karpathy's Autoresearch framework. The goal: let an LLM agent iteratively improve model performance while the author did weekend chores.

How Autoresearch Works

Autoresearch is a tight autonomous loop: hypothesize → edit → train → evaluate → commit or revert → repeat. The LLM agent modifies a single train.py file, reads a program.md for phased instructions, and keeps notes in a scratchpad.md as working memory. Each experiment is capped at ~5 minutes of wall-clock time to encourage fast iteration.

To prevent the agent from doing anything dangerous, the entire training loop was containerized with network access removed, and Claude Code's permissions were restricted to only editing two files and running a shell script.

Dataset & Model

The original eCLIP paper used medical X-ray datasets (no longer available), so the author switched to the Ukiyo-eVG dataset: ~11,000 Japanese woodblock prints with phrase-to-bounding-box annotations. Bounding boxes were converted to Gaussian heatmaps to guide the model's spatial attention, mirroring how radiologist eye-gaze data was used in the original paper.

Model: ViT-Small (22M) + DistilBERT (66M) + HeatmapProcessor ≈ 90M parameters total. Training: 800 steps (~3 minutes per run on an RTX 4090). Baseline mean rank on the validation set: 344.68.

Results

Over one Saturday, the agent ran 42 experiments, committing 13 and reverting 29. By the end of the day, the mean rank had dropped from 344.68 to 157.43 — a 54% reduction. A final full-dataset training run after the agent finished improved test scores even further.

Key Takeaways

  • Bug fix > everything: The biggest single gain (−113 mean rank) came from the agent spotting a temperature clamping bug in the original code — worth more than all architectural changes combined.
  • Great hyperparameter tuner: The agent methodically explored learning rates and projection dimensions, acting like a reasoning-augmented Optuna.
  • Diminishing returns in uncharted territory: Once the agent moved into moonshot architectural ideas, success rates dropped sharply — it was essentially throwing spaghetti at the wall.
  • Sandboxing matters: Toward the end the agent started making unexpected bash calls and occasionally gave up mid-loop. Full autonomy isn't quite there yet.
  • Commit-or-revert works: In a well-defined search space the loop is a surprisingly effective search strategy — but it needs guardrails when venturing into unknown unknowns.

Conclusion

The experiment demonstrated that an LLM agent can meaningfully drive ML research when the search space is structured. The first 90% went smoothly; the last 10% was a slog. The author ended the run after one day — it was movie night.
